Guest speaker

Kilian Valkhof

Rule of least power: choose the least powerful language suitable for a given purpose.

Summary

Today's lecture is was all about using HTML and CSS as much as you can without using too much JavaScript. Because the browser is built with HTML, it has many built in features that a lot of developers would try to built with Javascript. The benefits of using HTML over Javascript is that HTML also has built in accessibility features and do not require an endless list of packages and node modules. During this lecture, Kilian Valkhof talked about many different ways one can built features using only HTML and CSS.

  1. Custom toggle

    To create a toggle element one can use checkbox and design it into a toggle element

  2. Focus-visible

    Instead of using focus use focus-visible.

  3. Data list and options

    It's a way that when user types in a text field, options show while typing.

  4. Color picker

    The color picker tool built into HTML using type=“color”

  5. Smooth scrolling

    When user clicks on a link which leads to another heading of the page, it scrolls smoothly instead instance. “Scroll-behaviour: smooth;”

  6. Position sticky

    Use position sticky instead of fixed

What I learned

There are many different ways to create features that only use HTML instead of JavaScript. There are many benefits to using HTML and CSS over JavaScript. What I learned is to always use basic HTML and CSS to get something done. If not possible then use Javascript. There are many tags and features that i didn't know existed in HTML so it was certainly an eye opener. I would look out for doing things with HTML and CSS more whenever I can, and when not possible I would use JavaScript.
